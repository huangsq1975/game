# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Running the Game

No build step required — pure HTML/JS game.

```bash
# Start a local HTTP server (required for JSON config loading via fetch)
python3 -m http.server 8080
# Open http://localhost:8080
```

## Utility Scripts

```bash
# Regenerate config/Tilesets.json from the LibGDX atlas descriptor
python3 convert_atlas.py config/Tilesets.atlas config/Tilesets.json

# Export individual sprite PNGs from the atlas sheet (requires Pillow)
pip install Pillow
python3 split_tilesets.py
```

## Architecture

### Technology Stack
- **Phaser 3.60** — scene lifecycle, input, 2D canvas rendering
- **Pure WebGL** — 3D world-map sphere (no Three.js/Babylon.js)
- **Vanilla JS** — ~2,300 lines of game logic, all in `index.html`

### Key Files
| File | Purpose |
|------|---------|
| `index.html` | Entire game: Phaser scene, WebGL world map, UI |
| `units.js` | Declares global `UDEFS` object (populated from JSON at runtime) |
| `worldmap-items.js` | `WorldMapItems` class — landmark placement on 3D sphere |
| `config/game.json` | Master parameters (`GC` global): canvas size, hex grid, economy, fog, combat |
| `config/units.json` | Unit type definitions (cavalry / archers / infantry) |
| `config/map.json` | Tile assignments, fort positions, route arrows, base coords |
| `config/Tilesets.json` | 1,015 sprite frames (generated by `convert_atlas.py`) |

### Game Loop (inside `GameScene`)
1. **Preload** — atlas, map, unit configs loaded asynchronously; `game.json` loaded synchronously before Phaser init
2. **Create** — hex grid built, sprites placed, fog-of-war BFS initialized
3. **Update** — unit movement, combat resolution, fort capture, gold economy, AI decisions, rendering layers refreshed

### Coordinate System
Hexagonal grid uses **flat-top odd-q offset** coordinates. Neighbor lookups differ for even vs. odd columns. BFS from bases populates movement distance maps; `arrow` overrides in `map.json` redirect player units at specific hexes.

### Rendering Layers (depth order)
- Layers 0–4: hex fill/stroke, tile sprites
- Layers 5–9: fog-of-war polygons
- Layers 10–12: units (colored circles + glyph), HP bars
- Layer 15: UI overlay (buttons, timer, pause, settings)

### World Map (WebGL)
Separate render path from the Phaser game. Uses custom 4×4 matrix math for MVP transforms. Three render passes: stars (additive blend) → sphere (Phong shading) → atmosphere (rim glow). Terrain is procedurally generated via Canvas 2D + 3D Perlin fbm (7 octaves) with the same noise seed as `config/game.json`. Item placement rules are documented in `docs/worldmap-items-rules.md`.

### Combat & Economy
- Rock-paper-scissors unit advantages: cavalry > infantry > archers > cavalry
- Damage multiplier: 1.6× (advantage) / 0.6× (disadvantage) / 1.0× (neutral) with random variance
- Gold: 80 initial, 200 max, 10/sec base regen + 6/sec per owned fort
- AI rotates attack lanes every 7 seconds; difficulty affects spawn rate and unit mix

### Global State
| Variable | Contents |
|----------|----------|
| `GC` | All config from `game.json` |
| `UDEFS` | Unit type definitions from `units.json` |
| `mapData` | Parsed map: tiles, forts, arrows, bases |
| `units[]` | All live unit instances |
| `hexOwner` | Map of `"col,row"` → owner (`"player"` / `"enemy"`) |
| `dE`, `dP` | BFS fog-of-war distance maps for enemy and player |
